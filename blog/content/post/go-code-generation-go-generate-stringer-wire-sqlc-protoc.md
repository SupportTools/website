Code generation is one of Go's most underutilized productivity multipliers. By encoding repetitive, error-prone patterns into generators invoked through `go generate`, teams eliminate whole classes of bugs: mismatched enum string representations, hand-rolled SQL scanners that diverge from schema changes, and dependency injection wiring that breaks silently when a constructor signature changes. This guide covers a production-grade code generation stack built around `go generate`, demonstrating how to compose `stringer`, `enumer`, `wire`, `sqlc`, and `protoc` through a unified Makefile that works equally well locally and in CI.

# Go Code Generation with go generate

## Section 1: The go generate Mechanism

`go generate` is a simple tool with deep implications. When you run `go generate ./...`, it scans every `.go` file in the specified package tree for lines matching the pattern:

```
//go:generate <command>
```

It then executes each command in a shell, with the working directory set to the directory containing the source file. Several environment variables are set automatically:

| Variable      | Value                              |
|---------------|-------------------------------------|
| `$GOFILE`     | Filename of the source file         |
| `$GOPACKAGE`  | Package name of the source file     |
| `$GOARCH`     | Target architecture                 |
| `$GOOS`       | Target OS                           |
| `$GOROOT`     | Go installation root                |
| `$GOPATH`     | Go module path                      |
| `$DOLLAR`     | Literal `$` sign for escaping       |

A critical constraint: `go generate` is not run as part of `go build`. You must run it explicitly. This makes it a design-time tool, and the generated files should be committed to version control (unless you have a specific CI-based regeneration workflow).

### Project Layout Convention

Adopt a consistent layout for generated code:

```
internal/
  domain/
    status.go          # Defines type Status int
    status_string.go   # Generated by stringer
  db/
    queries/
      user.sql         # Hand-written SQL
    db.go              # Generated by sqlc
    models.go          # Generated by sqlc
    user.sql.go        # Generated by sqlc
  di/
    wire.go            # Wire provider declarations
    wire_gen.go        # Generated by wire
api/
  proto/
    v1/
      service.proto    # Hand-written protobuf definitions
  gen/
    v1/
      service.pb.go    # Generated by protoc
      service_grpc.pb.go  # Generated by protoc-gen-go-grpc
```

## Section 2: stringer — Enum String Representations

`stringer` generates `String()` methods for integer-based types, enabling them to satisfy `fmt.Stringer` and produce human-readable output in logs and error messages.

### Installation

```bash
go install golang.org/x/tools/cmd/stringer@latest
```

### Basic Usage

Define an integer type and annotate it:

```go
// internal/domain/status.go
package domain

//go:generate stringer -type=Status -linecomment

// Status represents the lifecycle state of an order.
type Status int

const (
    StatusUnknown    Status = iota // unknown
    StatusPending                  // pending
    StatusProcessing               // processing
    StatusCompleted                // completed
    StatusFailed                   // failed
    StatusCancelled                // cancelled
)
```

The `-linecomment` flag uses the comment after each constant as the string value rather than the constant name. This is essential when your internal names differ from the human-readable representation you want in logs.

Run the generator:

```bash
go generate ./internal/domain/
```

The output `status_string.go` contains:

```go
// Code generated by "stringer -type=Status -linecomment"; DO NOT EDIT.

package domain

import "strconv"

func _() {
    // An "invalid array index" compiler error signifies that the constant values have changed.
    var x [1]struct{}
    _ = x[StatusUnknown-0]
    _ = x[StatusPending-1]
    _ = x[StatusProcessing-2]
    _ = x[StatusCompleted-3]
    _ = x[StatusFailed-4]
    _ = x[StatusCancelled-5]
}

const _Status_name = "unknownpendingprocessingcompletedfailedcancelled"

var _Status_index = [...]uint8{0, 7, 14, 24, 33, 39, 48}

func (i Status) String() string {
    if i < 0 || i >= Status(len(_Status_index)-1) {
        return "Status(" + strconv.FormatInt(int64(i), 10) + ")"
    }
    return _Status_name[_Status_index[i]:_Status_index[i+1]]
}
```

The compile-time array size check is the critical safety mechanism: if you add a new constant and forget to re-run `go generate`, the build fails.

### Additional Flags

```bash
# Trim a prefix from all constant names
//go:generate stringer -type=Status -trimprefix=Status

# Output to a specific file
//go:generate stringer -type=Status -output=status_generated.go

# Multiple types in a single run
//go:generate stringer -type=Status,Priority,Category
```

## Section 3: enumer — Enhanced Enum Operations

`enumer` is a superset of `stringer` that also generates `ParseXxx`, `IsValidXxx`, `XxxValues()`, and optional JSON/SQL/YAML marshaling methods.

### Installation

```bash
go install github.com/dmarkham/enumer@latest
```

### Annotated Example

```go
// internal/domain/priority.go
package domain

//go:generate enumer -type=Priority -json -sql -yaml -linecomment -trimprefix=Priority

// Priority defines urgency for task scheduling.
type Priority int8

const (
    PriorityLow      Priority = iota // low
    PriorityMedium                   // medium
    PriorityHigh                     // high
    PriorityCritical                 // critical
)
```

Generated methods include:

```go
// PriorityString() — same as stringer
// PriorityValues() []Priority — returns all valid values
// PriorityStrings() []string — returns all string representations
// PriorityP.IsAPriority() bool — validates enum membership
// ParsePriority(s string) (Priority, error) — parses from string
// (p Priority) MarshalJSON() ([]byte, error) — JSON marshaling
// (p *Priority) UnmarshalJSON(b []byte) error — JSON unmarshaling
// (p Priority) Value() (driver.Value, error) — database/sql driver
// (p *Priority) Scan(value interface{}) error — database/sql scanner
```

This eliminates hand-written JSON and SQL marshaling for every enum in your codebase.

### Integration in Tests

```go
// internal/domain/priority_test.go
package domain_test

import (
    "testing"
    "github.com/yourorg/yourapp/internal/domain"
)

func TestPriorityRoundTrip(t *testing.T) {
    for _, p := range domain.PriorityValues() {
        got, err := domain.ParsePriority(p.String())
        if err != nil {
            t.Fatalf("ParsePriority(%q) error: %v", p.String(), err)
        }
        if got != p {
            t.Errorf("round-trip failed: got %v, want %v", got, p)
        }
    }
}
```

## Section 4: wire — Compile-Time Dependency Injection

Google's `wire` tool generates dependency injection code at compile time, eliminating runtime reflection and making the dependency graph visible in source code.

### Installation

```bash
go install github.com/google/wire/cmd/wire@latest
```

### Provider Functions

Providers are ordinary Go functions that return a type and optionally an error:

```go
// internal/di/providers.go
package di

import (
    "context"
    "database/sql"
    "fmt"

    "github.com/yourorg/yourapp/internal/config"
    "github.com/yourorg/yourapp/internal/db"
    "github.com/yourorg/yourapp/internal/service"
    "github.com/yourorg/yourapp/internal/repository"
    _ "github.com/lib/pq"
)

// ProvideConfig loads application configuration.
func ProvideConfig() (*config.Config, error) {
    return config.LoadFromEnv()
}

// ProvideDB creates a database connection pool.
func ProvideDB(cfg *config.Config) (*sql.DB, error) {
    dsn := fmt.Sprintf(
        "host=%s port=%d user=%s password=%s dbname=%s sslmode=%s",
        cfg.DB.Host, cfg.DB.Port, cfg.DB.User, cfg.DB.Password,
        cfg.DB.Name, cfg.DB.SSLMode,
    )
    pool, err := sql.Open("postgres", dsn)
    if err != nil {
        return nil, fmt.Errorf("open database: %w", err)
    }
    pool.SetMaxOpenConns(cfg.DB.MaxOpenConns)
    pool.SetMaxIdleConns(cfg.DB.MaxIdleConns)
    pool.SetConnMaxLifetime(cfg.DB.ConnMaxLifetime)
    return pool, nil
}

// ProvideQueries creates a sqlc Queries struct from a database connection.
func ProvideQueries(db *sql.DB) *db.Queries {
    return db.New(db)
}

// ProvideUserRepository creates a user repository.
func ProvideUserRepository(q *db.Queries) *repository.UserRepository {
    return repository.NewUserRepository(q)
}

// ProvideUserService creates a user service.
func ProvideUserService(repo *repository.UserRepository) *service.UserService {
    return service.NewUserService(repo)
}
```

### Wire Sets and Injectors

Group providers into reusable sets:

```go
// internal/di/wire.go
//go:build wireinject
// +build wireinject

package di

import (
    "github.com/google/wire"
    "github.com/yourorg/yourapp/internal/service"
)

// InfrastructureSet bundles all infrastructure providers.
var InfrastructureSet = wire.NewSet(
    ProvideConfig,
    ProvideDB,
    ProvideQueries,
)

// RepositorySet bundles all repository providers.
var RepositorySet = wire.NewSet(
    ProvideUserRepository,
)

// ServiceSet bundles all service providers.
var ServiceSet = wire.NewSet(
    ProvideUserService,
)

// InitializeApplication wires all dependencies and returns the root service.
// This function body is replaced by generated code; the panic is never reached.
func InitializeApplication() (*service.UserService, error) {
    wire.Build(
        InfrastructureSet,
        RepositorySet,
        ServiceSet,
    )
    return nil, nil
}
```

The `//go:build wireinject` constraint prevents the stub from being compiled normally. Run wire to generate the implementation:

```bash
cd internal/di && wire
```

Wire generates `wire_gen.go`:

```go
// Code generated by Wire. DO NOT EDIT.

//go:build !wireinject
// +build !wireinject

package di

import (
    "github.com/yourorg/yourapp/internal/service"
)

// Injectors from wire.go:

func InitializeApplication() (*service.UserService, error) {
    cfg, err := ProvideConfig()
    if err != nil {
        return nil, err
    }
    db, err := ProvideDB(cfg)
    if err != nil {
        return nil, err
    }
    queries := ProvideQueries(db)
    userRepository := ProvideUserRepository(queries)
    userService := ProvideUserService(userRepository)
    return userService, nil
}
```

### Wire for Tests

Wire also helps with test fixtures:

```go
// internal/di/wire_test.go
//go:build wireinject

package di

import (
    "github.com/google/wire"
    "github.com/yourorg/yourapp/internal/service"
    "github.com/yourorg/yourapp/internal/testutil"
)

var TestInfrastructureSet = wire.NewSet(
    testutil.ProvideTestDB,     // Uses testcontainers
    ProvideQueries,
)

func InitializeTestApplication() (*service.UserService, error) {
    wire.Build(
        TestInfrastructureSet,
        RepositorySet,
        ServiceSet,
    )
    return nil, nil
}
```

## Section 5: sqlc — Type-Safe SQL from Schema

`sqlc` compiles annotated SQL queries into type-safe Go code. It reads your schema, validates queries against it, and generates struct types, scanner methods, and function wrappers.

### Installation

```bash
go install github.com/sqlc-dev/sqlc/cmd/sqlc@latest
```

### Configuration

```yaml
# sqlc.yaml
version: "2"
sql:
  - engine: "postgresql"
    queries: "internal/db/queries/"
    schema: "internal/db/schema/"
    gen:
      go:
        package: "db"
        out: "internal/db"
        emit_json_tags: true
        emit_db_tags: true
        emit_prepared_queries: true
        emit_interface: true
        emit_exact_table_names: false
        emit_empty_slices: true
        json_tags_case_style: "snake"
        overrides:
          # Map PostgreSQL UUID to google/uuid
          - db_type: "uuid"
            go_type:
              import: "github.com/google/uuid"
              type: "UUID"
          # Map PostgreSQL TIMESTAMPTZ to time.Time
          - db_type: "timestamptz"
            go_type: "time.Time"
          # Map custom domain types
          - db_type: "public.status"
            go_type:
              import: "github.com/yourorg/yourapp/internal/domain"
              type: "Status"
```

### Schema Definition

```sql
-- internal/db/schema/001_users.sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

CREATE TABLE users (
    id          UUID        PRIMARY KEY DEFAULT uuid_generate_v4(),
    email       TEXT        NOT NULL UNIQUE,
    name        TEXT        NOT NULL,
    status      INT2        NOT NULL DEFAULT 0,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_status ON users(status);

-- internal/db/schema/002_orders.sql
CREATE TABLE orders (
    id          UUID        PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id     UUID        NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    total       NUMERIC(12,2) NOT NULL,
    status      INT2        NOT NULL DEFAULT 0,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_orders_user_id ON orders(user_id);
CREATE INDEX idx_orders_status ON orders(status);
```

### Annotated Queries

```sql
-- internal/db/queries/users.sql

-- name: GetUser :one
SELECT * FROM users
WHERE id = $1 LIMIT 1;

-- name: GetUserByEmail :one
SELECT * FROM users
WHERE email = $1 LIMIT 1;

-- name: ListUsers :many
SELECT * FROM users
WHERE status = $1
ORDER BY created_at DESC
LIMIT $2 OFFSET $3;

-- name: CreateUser :one
INSERT INTO users (email, name, status)
VALUES ($1, $2, $3)
RETURNING *;

-- name: UpdateUserStatus :one
UPDATE users
SET status = $2, updated_at = NOW()
WHERE id = $1
RETURNING *;

-- name: DeleteUser :exec
DELETE FROM users WHERE id = $1;

-- name: CountUsersByStatus :one
SELECT COUNT(*) FROM users WHERE status = $1;
```

Annotate the source file to trigger generation:

```go
// internal/db/generate.go
package db

//go:generate sqlc generate
```

Run:

```bash
go generate ./internal/db/
```

Generated code example for `GetUser`:

```go
// internal/db/users.sql.go (generated)
const getUser = `-- name: GetUser :one
SELECT id, email, name, status, created_at, updated_at FROM users
WHERE id = $1 LIMIT 1
`

func (q *Queries) GetUser(ctx context.Context, id uuid.UUID) (User, error) {
    row := q.db.QueryRowContext(ctx, getUser, id)
    var i User
    err := row.Scan(
        &i.ID,
        &i.Email,
        &i.Name,
        &i.Status,
        &i.CreatedAt,
        &i.UpdatedAt,
    )
    return i, err
}
```

## Section 6: protoc and buf — Protocol Buffer Generation

For gRPC services, `buf` replaces the raw `protoc` command with dependency management, linting, and breaking change detection.

### Installation

```bash
# Install buf
go install github.com/bufbuild/buf/cmd/buf@latest

# Install protoc plugins
go install google.golang.org/protobuf/cmd/protoc-gen-go@latest
go install google.golang.org/grpc/cmd/protoc-gen-go-grpc@latest
go install github.com/grpc-ecosystem/grpc-gateway/v2/protoc-gen-grpc-gateway@latest
go install github.com/grpc-ecosystem/grpc-gateway/v2/protoc-gen-openapiv2@latest
```

### buf Configuration

```yaml
# buf.yaml
version: v2
modules:
  - path: api/proto
deps:
  - buf.build/googleapis/googleapis
  - buf.build/grpc-ecosystem/grpc-gateway
lint:
  use:
    - DEFAULT
  except:
    - FIELD_LOWER_SNAKE_CASE
  ignore:
    - api/proto/v1/internal
breaking:
  use:
    - FILE
  ignore:
    - api/proto/v1/experimental
```

```yaml
# buf.gen.yaml
version: v2
managed:
  enabled: true
  override:
    - file_option: go_package_prefix
      value: github.com/yourorg/yourapp/api/gen
plugins:
  - remote: buf.build/protocolbuffers/go
    out: api/gen
    opt:
      - paths=source_relative
  - remote: buf.build/grpc/go
    out: api/gen
    opt:
      - paths=source_relative
      - require_unimplemented_servers=false
  - remote: buf.build/grpc-ecosystem/gateway
    out: api/gen
    opt:
      - paths=source_relative
      - generate_unbound_methods=true
  - remote: buf.build/grpc-ecosystem/openapiv2
    out: api/gen/openapi
    opt:
      - output_format=json
```

### Proto Service Definition

```protobuf
// api/proto/v1/user_service.proto
syntax = "proto3";

package api.v1;

import "google/api/annotations.proto";
import "google/protobuf/timestamp.proto";
import "protoc-gen-openapiv2/options/annotations.proto";

option go_package = "github.com/yourorg/yourapp/api/gen/v1;apiv1";

option (grpc.gateway.protoc_gen_openapiv2.options.openapiv2_swagger) = {
  info: {
    title: "User Service API";
    version: "1.0";
  };
};

service UserService {
  rpc GetUser(GetUserRequest) returns (GetUserResponse) {
    option (google.api.http) = {
      get: "/v1/users/{user_id}"
    };
  }
  rpc CreateUser(CreateUserRequest) returns (CreateUserResponse) {
    option (google.api.http) = {
      post: "/v1/users"
      body: "*"
    };
  }
  rpc ListUsers(ListUsersRequest) returns (ListUsersResponse) {
    option (google.api.http) = {
      get: "/v1/users"
    };
  }
}

message User {
  string user_id    = 1;
  string email      = 2;
  string name       = 3;
  int32  status     = 4;
  google.protobuf.Timestamp created_at = 5;
  google.protobuf.Timestamp updated_at = 6;
}

message GetUserRequest {
  string user_id = 1;
}

message GetUserResponse {
  User user = 1;
}

message CreateUserRequest {
  string email = 1;
  string name  = 2;
}

message CreateUserResponse {
  User user = 1;
}

message ListUsersRequest {
  int32 status   = 1;
  int32 page     = 2;
  int32 per_page = 3;
}

message ListUsersResponse {
  repeated User users = 1;
  int64 total         = 2;
}
```

Annotate and run:

```go
// api/proto/generate.go
package proto

//go:generate buf generate
```

```bash
go generate ./api/proto/
```

## Section 7: Makefile Orchestration

Centralizing all generators in a Makefile makes CI and local development identical:

```makefile
# Makefile

SHELL := /bin/bash
.DEFAULT_GOAL := help

# Tool versions
STRINGER_VERSION  := v0.20.0
ENUMER_VERSION    := latest
WIRE_VERSION      := v0.6.0
SQLC_VERSION      := v1.26.0
BUF_VERSION       := v1.30.0

# Go module path
MODULE := github.com/yourorg/yourapp

# Generated file marker (used in .gitattributes to mark as generated)
GENERATED_MARKER := "// Code generated"

.PHONY: tools
tools: ## Install all code generation tools
	@echo "Installing code generation tools..."
	go install golang.org/x/tools/cmd/stringer@$(STRINGER_VERSION)
	go install github.com/dmarkham/enumer@$(ENUMER_VERSION)
	go install github.com/google/wire/cmd/wire@$(WIRE_VERSION)
	go install github.com/sqlc-dev/sqlc/cmd/sqlc@$(SQLC_VERSION)
	go install github.com/bufbuild/buf/cmd/buf@$(BUF_VERSION)
	go install google.golang.org/protobuf/cmd/protoc-gen-go@latest
	go install google.golang.org/grpc/cmd/protoc-gen-go-grpc@latest
	go install github.com/grpc-ecosystem/grpc-gateway/v2/protoc-gen-grpc-gateway@latest
	go install github.com/grpc-ecosystem/grpc-gateway/v2/protoc-gen-openapiv2@latest

.PHONY: generate
generate: generate-enums generate-wire generate-sqlc generate-proto ## Run all code generators

.PHONY: generate-enums
generate-enums: ## Run stringer and enumer for enum types
	@echo "Generating enum types..."
	go generate ./internal/domain/...
	@echo "Done generating enum types."

.PHONY: generate-wire
generate-wire: ## Run wire dependency injection generator
	@echo "Generating wire dependency injection..."
	cd internal/di && wire
	@echo "Done generating wire."

.PHONY: generate-sqlc
generate-sqlc: ## Run sqlc SQL code generator
	@echo "Generating sqlc SQL bindings..."
	sqlc generate
	@echo "Done generating sqlc."

.PHONY: generate-proto
generate-proto: ## Run buf protobuf generator
	@echo "Generating protobuf code..."
	buf lint
	buf generate
	@echo "Done generating protobuf code."

.PHONY: generate-check
generate-check: ## Verify generated code is up-to-date (for CI)
	@echo "Checking generated code is up-to-date..."
	$(MAKE) generate
	@if [ -n "$$(git diff --name-only)" ]; then \
		echo "Generated code is out of date. Run 'make generate' and commit the changes."; \
		git diff --stat; \
		exit 1; \
	fi
	@echo "Generated code is up-to-date."

.PHONY: proto-lint
proto-lint: ## Lint proto files
	buf lint

.PHONY: proto-breaking
proto-breaking: ## Check for breaking changes in proto files
	buf breaking --against '.git#branch=main'

.PHONY: sqlc-verify
sqlc-verify: ## Verify sqlc queries compile against schema
	sqlc compile

.PHONY: clean-generated
clean-generated: ## Remove all generated files (use with caution)
	@echo "Removing generated files..."
	find . -name "*.pb.go" -not -path "./vendor/*" -delete
	find . -name "*_grpc.pb.go" -not -path "./vendor/*" -delete
	find . -name "*_string.go" -not -path "./vendor/*" -delete
	find . -name "wire_gen.go" -not -path "./vendor/*" -delete
	find . -name "*.sql.go" -not -path "./vendor/*" -delete
	find . -path "./internal/db/db.go" -delete
	find . -path "./internal/db/models.go" -delete
	@echo "Generated files removed."

.PHONY: help
help: ## Display this help message
	@awk 'BEGIN {FS = ":.*##"; printf "\nUsage:\n  make \033[36m<target>\033[0m\n\nTargets:\n"} \
	/^[a-zA-Z_-]+:.*?##/ { printf "  \033[36m%-20s\033[0m %s\n", $$1, $$2 }' $(MAKEFILE_LIST)
```

## Section 8: CI Integration

### GitHub Actions Workflow

```yaml
# .github/workflows/codegen-check.yml
name: Code Generation Check

on:
  pull_request:
    paths:
      - "internal/domain/*.go"
      - "internal/db/schema/**"
      - "internal/db/queries/**"
      - "internal/di/*.go"
      - "api/proto/**"
      - "sqlc.yaml"
      - "buf.yaml"
      - "buf.gen.yaml"

jobs:
  codegen-check:
    name: Verify Generated Code
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version: "1.23"
          cache: true

      - name: Cache tools
        uses: actions/cache@v4
        with:
          path: ~/go/bin
          key: go-tools-${{ hashFiles('Makefile') }}

      - name: Install tools
        run: make tools

      - name: Run code generation
        run: make generate

      - name: Check for uncommitted changes
        run: |
          if [ -n "$(git diff --name-only)" ]; then
            echo "::error::Generated code is out of date."
            echo "The following files have changed:"
            git diff --name-only
            echo ""
            echo "Run 'make generate' locally and commit the generated files."
            exit 1
          fi
          echo "All generated code is up-to-date."

      - name: Verify proto linting
        run: make proto-lint

      - name: Check for breaking proto changes
        run: make proto-breaking
        continue-on-error: true  # Non-blocking for feature branches
```

## Section 9: Advanced Patterns

### Custom Generators

Write a custom generator for repetitive patterns in your own codebase:

```go
// tools/gen-validators/main.go
package main

import (
    "bytes"
    "flag"
    "go/ast"
    "go/parser"
    "go/token"
```

## Section 10: Versioning and Maintenance

### Detecting Stale Generated Files

Add a CI lint check using a script:

```bash
#!/usr/bin/env bash
# scripts/check-generated.sh
set -euo pipefail

echo "Checking generated files are marked correctly..."

GENERATED_FILES=$(git ls-files | xargs grep -l "Code generated" 2>/dev/null || true)
UNMARKED_FILES=()

for f in $GENERATED_FILES; do
    # Ensure file has DO NOT EDIT marker
    if ! grep -q "DO NOT EDIT" "$f" 2>/dev/null; then
        UNMARKED_FILES+=("$f")
    fi
done

if [ ${#UNMARKED_FILES[@]} -gt 0 ]; then
    echo "ERROR: The following generated files are missing 'DO NOT EDIT' marker:"
    printf '  %s\n' "${UNMARKED_FILES[@]}"
    exit 1
fi

echo "All generated files are correctly marked."
```

### Upgrading Tool Versions

Use a dedicated version file:

```makefile
# Makefile.versions (included by main Makefile)
STRINGER_VERSION  := v0.20.0
ENUMER_VERSION    := v1.5.10
WIRE_VERSION      := v0.6.0
SQLC_VERSION      := v1.26.0
BUF_VERSION       := v1.30.0
MOCKGEN_VERSION   := v0.4.0
```

Update versions via Dependabot or Renovate:

```yaml
# .github/renovate.json
{
  "$schema": "https://docs.renovatebot.com/renovate-schema.json",
  "extends": ["config:base"],
  "regexManagers": [
    {
      "fileMatch": ["^Makefile\\.versions$"],
      "matchStrings": [
        "(?<depName>[A-Z_]+_VERSION)\\s*:=\\s*(?<currentValue>v[\\d\\.]+)"
      ],
      "datasourceTemplate": "go",
      "packageNameTemplate": "{{#if (equals depName 'STRINGER_VERSION')}}golang.org/x/tools/cmd/stringer{{/if}}"
    }
  ]
}
```

The complete code generation stack — stringer for enum strings, enumer for full enum marshaling, wire for dependency injection, sqlc for type-safe SQL, and buf for protobufs — eliminates the most error-prone categories of boilerplate in Go enterprise codebases. When all generators are wired through `go generate` directives and orchestrated by a single `make generate` target, both local development and CI operate from a single source of truth.
